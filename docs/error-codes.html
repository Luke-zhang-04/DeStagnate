Errors Reference

In an effort to reduce bundle size, error messages have been moved here.

Code 1

Error code 1

1 Parameter: component parent (parent).

Raised in: 

DeStagnate.Component

"Avoid using this ${parent} as element parent, as all elements within this ${parent} will be removed on re-render. To disable this, pass in true as a third constructor parameter"

The parent you used has content inside of it, which will be overwritten when this component updates. To disable this error, pass in 

true
as the third parameter to the component constructor

Code 2

Error code 2

0 Parameters.

Raised in: 

DeStagnate.Component

"Do not mutate state directly. Use setState instead."

You likely mutated state by doing 

this.state = {}
somewhere that is not your constructor. Use the 
this.setState({})
method instead. To disable this, invoke 
this.disableStrict()
.

Code 3

Error code 3

0 Parameters.

Raised in: 

DeStagnate.Component

"Parent is not defined. Set parent with the parent setter or set it during mounting."

You have not defined a parent element for your component. Define a parent on instantiation (in the constructor), or when you call the 

this.mount()
method (pass as param).

Code 4

Warn code 4

0 Parameters.

Raised in: 

DeStagnate.Component

"No parent was set. Component unmounted from nothing."

Simillar to code 3, but is a warning. If you got code 4, you've unmounted a component that had no parent. Since no parent was necessary, no error is raised.

Code 5

Error code 5

0 Parameters.

Raised in: 

DeStagnate.Component

"Expected render method to be included in component class, no render method found, or render returned an empty array"

Your component class needs to implement the 

this.render()
method. If you're using TypeScript, you might get the message "Non-abstract class 
myComponent
does not implement inherited abstract member 
render
from class 
DeStagnate<Record<string, unknown>, Record<string, unknown>>.
"

Code 6

Warning code 6

2 Parameters. newKey: new key passed in through setState, and existingKeys: already existing keys in state.

Raised in: 

DeStagnate.Component

"New key ${newKey} should not be set with setState, which has keys ${existingKeys}. Declare all state variables in constructor as a best practice. Did you misspell a key?"

When you called 

this.setState()
, you added a key to state that did not exist before. For best pracices, every key of your state should've been declared in your 
constructor
. There is also a good chance you misspelled a key. To supress this warning, invoke 
this.disableStrict()
.

Code 7

Warning code 7

2 Parameters: keyType: type of key from props, and keyName: name of key from props.

Raised in: 

DeStagnate.createElement
, and other simillar functions.

"Type of key ${keyType} for ${keyName} does not fit in as a valid prop type, and will me skipped.

When you called createElement, some of your props weren't of a compatible type, and fell through.

Code 8

Error code 8

0 Parameters.

Raised in: 

DeStagnate.createElement
, and other simillar functions.

"Cannot use non-HTMLElement as component parent"

When you passed in a DeStagnate component as a child, it's parent was not an HTMLElement.
