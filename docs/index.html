No Virtual DOM

Virtual DOM is unecessary overhead for static sites looking to add a dynamic component. This makes for a super small bundle size.

See why Svelte thinks virtual DOM is pure overhead.

Write Less Code

Leave vanilla DOM manipulation and stringed HTML behind and work with component and element creation easily. DeStagnate provides tools for nested componentry, JSX state management, and more.

See some examples

Component-Based

DeStagnate vs Vanilla DOM

Here's a side-by-side comparison of DeStagnate vs Vanilla DOM Manipulation

                    
const className = "container bg-light text-dark py-1 br-1 my-1"

// Structured like your component/element
document.getElementById("compare")?.children[0]?.appendChild(
    createElement("div", {class: className},
        createElement("p", {class: "m-0 p-0"}, "Hello World!")
    )
)
                    
                
                    
// Lack of structure

const helloWorldContainer = document.createElement("div"),
    helloWorld = document.createElement("p"),
    className = "container bg-light text-dark py-1 br-1 my-1"

helloWorldContainer.setAttribute("class", className)
helloWorld.setAttribute("class", "m-0 p-0")
helloWorld.innerText = "Hello World!"

helloWorldContainer.appendChild(helloWorld)

document.getElementById("compare")
    ?.children[1]
    ?.appendChild(helloWorldContainer)
